
**Назначение файлов:**
- `exceptions.py` – определяет собственные исключения для предметной области.
- `storage.py` – сериализация/десериализация объектов `Athlete`, `CompetitiveAthlete`, `RecreationalAthlete` в/из JSON.
- `app.py` – класс `App`, который управляет коллекцией, загружает данные при старте, сохраняет при изменениях.
- `cli.py` – функции для печати таблицы, ввода данных спортсмена, основного цикла меню.
- `main.py` – запуск цикла CLI.

## 3. Описание CLI

### Пункты меню
Приложение предоставляет следующие возможности (доступные через нумерованное меню):

1. **Добавить спортсмена** – запрашивает тип (обычный / соревновательный / любитель), все характеристики, валидирует ввод.
2. **Показать всех спортсменов** – выводит таблицу с полями: имя, вес, рост, рекорд, активность, здоровье, уровень. Для специальных типов показываются дополнительные сведения (победы/рейтинг или любимое занятие/удовольствие).
3. **Найти спортсмена по имени** – ищет и показывает результат.
4. **Фильтрация** – подменю: только активные, только здоровые, только соревновательные, только любители, рекорд выше заданного значения.
5. **Сортировка** – подменю: по имени, весу, рекорду, рейтингу (для соревновательных). Возможен порядок по возрастанию/убыванию.
6. **Удалить спортсмена** – запрашивает имя, требует подтверждения (y/n).
0. **Выход** – сохраняет данные в файл `athletes.json`.

### Обработка ошибок ввода
- При выборе пункта меню используется `try/except` для преобразования ввода в число; при ошибке выводится сообщение и меню повторяется.
- При вводе числовых значений (вес, рост, рекорд, рейтинг, уровень удовольствия) перехватывается `ValueError` и выводится понятная ошибка, пользователь возвращается к меню.
- Проверка на пустое имя.
- При добавлении дубликата (спортсмен с таким же именем) выбрасывается `DuplicateItemError` и сообщается об ошибке.
- При удалении несуществующего спортсмена – `ItemNotFoundException`.

### Сохранение и загрузка 
- При запуске приложения данные автоматически загружаются из файла `athletes.json` (если файл отсутствует – создаётся пустая коллекция).
- После каждой операции, изменяющей коллекцию (добавление, удаление, сортировка), вызывается `_save_data()`, которая сохраняет текущее состояние в JSON.
- При выходе из программы также вызывается сохранение.
