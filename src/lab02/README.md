\#\#\# Цель работы
 * Научиться работать с коллекциями объектов.
*  Понять разницу между моделью сущности (Athlete) и контейнером объектов (AthleteCollection).
*  Реализовать собственный контейнерный класс с методами добавления, удаления, поиска, сортировки, фильтрации.
* Освоить итерацию по объектам и магические методы (\_\_len\_\_, \_\_iter\_\_, \_\_getitem\_\_).

\-\-\-

Реализованный класс коллекции
AthleteCollection – хранит объекты Athlete во внутреннем списке self.\_items.  
Все методы коллекции проверяют тип добавляемого объекта и запрещают дубликаты (по равенству Athlete).

\-\-\-

### Основные методы (оценка 3)

| Метод | Описание |
|-------|----------|
| add(athlete) | Добавляет спортсмена, проверяет тип и отсутствие дубликата |
| remove(athlete) | Удаляет спортсмена по объекту |
| get\_all() | Возвращает копию списка всех спортсменов |
| \_\_len\_\_() | Возвращает количество элементов |
| \_\_iter\_\_() | Позволяет использовать for a in collection |

\-\-\-

### Поиск (оценка 4)

| Метод | Описание |
|-------|----------|
| find\_by\_name(name) | Возвращает список спортсменов с точным совпадением имени |
| find\_by\_record(min, max) | По диапазону личного рекорда |
| find\_by\_health\_status(status) | По состоянию здоровья (healthy, injured, recovering) |
| find\_by\_training\_level(level) | По уровню подготовки (beginner, intermediate, advanced, professional) |
| find\_by\_active(active) | По активности (активные / неактивные) |

\-\-\-

### Сортировка (оценка 5)

| Метод | Описание |
|-------|----------|
| sort(key, reverse) | Универсальная сортировка с ключом (например, по имени, весу, рекорду) |

Примеры ключей:  
\- lambda a: a.name – по имени  
\- lambda a: a.weight – по весу  
\- lambda a: a.personal\_record – по рекорду

\-\-\-

### Фильтрация (логические операции, возвращают новую коллекцию)

| Метод | Описание |
|-------|----------|
| filter\_active() | Только активные спортсмены |
| filter\_health\_status(status) | По статусу здоровья |
| filter\_training\_level(level) | По уровню подготовки |
| filter\_min\_record(min\_record) | Рекорд не ниже заданного |

Все методы возвращают новый объект AthleteCollection, не изменяя исходный.

\-\-\-

### Разница между моделью сущности и контейнером

| Сущность (Athlete) | Контейнер (AthleteCollection) |
|--------------------|--------------------------------|
| Хранит данные одного спортсмена (имя, вес, рост, рекорд, состояния) | Хранит группу спортсменов |
| Содержит бизнес\-методы (bmi(), set\_record(), activate()) | Содержит методы управления группой (добавление, удаление, поиск, сортировка) |
| Отвечает за валидацию своих атрибутов | Отвечает за целостность коллекции (типы, дубликаты) |
| Имеет состояние (активен/неактивен, здоровье, мораль) | Не имеет собственного состояния, только список элементов |
