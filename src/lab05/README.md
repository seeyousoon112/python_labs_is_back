# Лабораторная работа №5 – Функции как аргументы. Стратегии и делегаты.

## 1. Цель работы

Изучение передачи функций как аргументов в другие функции и методы; применение встроенных функций высшего порядка `map`, `filter`, `sorted`; реализация паттерна «Стратегия» с использованием callable-объектов; интеграция функционального стиля с объектно-ориентированным кодом из предыдущих лабораторных работ.

## 2. Реализованные функции и стратегии

### Стратегии сортировки (key-функции)
- `by_name` – сортировка по имени спортсмена.
- `by_weight` – сортировка по весу.
- `by_record` – сортировка по личному рекорду.
- `by_rating` – сортировка по рейтингу (для `CompetitiveAthlete`).
- `by_enjoyment` – сортировка по уровню удовольствия (для `RecreationalAthlete`).

### Функции-фильтры (предикаты)
- `is_active` – оставляет только активных спортсменов.
- `is_healthy` – оставляет только здоровых.
- `is_competitive` – оставляет только соревновательных спортсменов.
- `is_recreational` – оставляет только любителей.

### Функции высшего порядка и фабрики
- `has_record_above(threshold)` – фабрика, возвращающая фильтр: рекорд > threshold.
- `make_weight_filter(max_weight)` – фабрика, возвращающая фильтр: вес ≤ max_weight.
- `apply_discount(discount)` – фабрика, возвращающая функцию для уменьшения рекорда на discount%.
- `increase_morale(delta)` – функция увеличения морали.

### Паттерн «Стратегия» и callable-объекты
В модуль `strategies.py` вынесены все стратегии (key-функции, предикаты, фабрики).  
Коллекция `FunctionalAthleteCollection` (наследник `AthleteCollection` из ЛР‑2) предоставляет методы:
- `sort_by(key_func, reverse)` – принимает стратегию сортировки и сортирует коллекцию.
- `filter_by(predicate)` – принимает стратегию фильтрации и возвращает новую коллекцию.
- `apply(func)` – принимает callable-объект и применяет его ко всем элементам.

Это позволяет легко заменять стратегии без изменения кода коллекции.

## 4. Вывод

В ходе лабораторной работы №5 изучены и применены на практике:

- **Передача функций как аргументов** – в методы `sort_by`, `filter_by`, `apply` и во встроенные функции `map`, `filter`, `sorted`.
- **Лямбда-выражения** – для создания небольших анонимных функций (например, `lambda a: a.weight`).
- **Функции высшего порядка и замыкания** – реализованы фабрики `has_record_above`, `make_weight_filter`, `apply_discount`.
- **Паттерн «Стратегия»** – вынесение варьирующегося поведения (критерии сортировки, условия фильтрации) в отдельные функции и их подстановка через единый интерфейс. Метод `apply` демонстрирует передачу произвольной callable-стратегии для обработки всех элементов коллекции.
