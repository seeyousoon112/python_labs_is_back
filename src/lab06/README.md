# Лабораторная работа №6 – Generics и typing

## 1. Цель работы

Освоить систему аннотаций типов в Python (`typing`), научиться создавать обобщённые (generic) классы с помощью `TypeVar` и `Generic`, понять концепцию структурной типизации через `typing.Protocol`. На практике применить типизацию к существующим классам и коллекции из предыдущих лабораторных работ.

## 2. Описание реализованных типов и контейнеров

### Generic-класс `TypedCollection[T]`

Реализован в файле `container.py`. Это обобщённая версия коллекции из ЛР‑2, которая параметризована типом хранимых элементов `T`. Основные методы:

- `add(item: T) -> None` – добавляет элемент типа `T`.
- `remove(item: T) -> None` – удаляет элемент.
- `get_all() -> list[T]` – возвращает копию списка.
- `__len__()`, `__iter__()`, `__getitem__()`, `remove_at()`, `sort()`.
- `find(predicate: Callable[[T], bool]) -> Optional[T]`
- `filter(predicate: Callable[[T], bool]) -> list[T]`
- `map(transform: Callable[[T], R]) -> list[R]` – демонстрирует работу со вторым параметром типа `R`.

**Зачем:** Позволяет создавать типобезопасные коллекции для любых объектов, не дублируя код.

### TypeVar и ограничения (bound)

Использованы:

- `T = TypeVar('T')` – основной параметр типа коллекции.
- `R = TypeVar('R')` – для метода `map` (тип результата преобразования).
- `D = TypeVar('D', bound=Displayable)` – для коллекций, содержащих объекты, удовлетворяющие протоколу `Displayable`.
- `S = TypeVar('S', bound=Scorable)` – для коллекций, содержащих объекты, удовлетворяющие протоколу `Scorable`.

Ограничение `bound` гарантирует, что переданный тип имеет все методы, требуемые протоколом.

### Протоколы 

Определены два протокола в `container.py`:

- `class Displayable(Protocol): def display(self) -> str: ...`
- `class Scorable(Protocol): def score(self) -> float: ...`

Этим протоколам **неявно** удовлетворяют классы `Athlete`, `CompetitiveAthlete`, `RecreationalAthlete`, так как в них добавлены методы `display()` и `score()` (без явного наследования от протоколов). Благодаря этому объекты этих классов могут быть помещены в `TypedCollection[Displayable]` или `TypedCollection[Scorable]`.

## 4. Вывод

В ходе лабораторной работы №6 изучены и применены на практике:

- **Аннотации типов** – добавлены к существующим классам (`Athlete`), что улучшает читаемость и позволяет статическим анализаторам находить ошибки.
- **Generic и TypeVar** – создан обобщённый класс `TypedCollection[T]`, который можно использовать с любым типом элементов, сохраняя типобезопасность. Показана работа с несколькими параметрами типа (`T` и `R` в `map`).
- **Протоколы (`Protocol`)** – реализована структурная типизация: объекты подходят под протокол, если имеют нужные методы, без явного наследования. Это повышает гибкость кода.
- **Типизация как документация** – аннотации типов делают код самодокументированным, а IDE получает возможность подсказывать методы и проверять корректность вызовов.
